# Quickstart: 해설지 레이아웃 개선

**Feature**: 001-explanation-layout
**Date**: 2025-12-23

## 개발 환경 설정

### 1. 의존성 설치

```bash
npm install
```

### 2. 개발 서버 실행

```bash
npm run dev
```

브라우저에서 http://localhost:3000 접속

## 기능 테스트 가이드

### 1. 해설지 뷰 모드 진입

1. 문제집 모드에서 TSV 데이터 입력
2. "해설지" 탭 클릭하여 뷰 모드 전환
3. 해설지 레이아웃 확인

### 2. Quick Answer 표 확인

1. 해설지 상단에 정답표 표시 확인
2. 7열 형식, 문제 수에 따라 행 자동 확장 확인
3. 정답이 숫자(1-5) 형식으로 표시되는지 확인

### 3. 유형별 해설 레이아웃 확인

| 유형 | 확인 항목 |
|------|----------|
| 어휘(동의어) | 밑줄 단어, 정답, 동의어 목록 |
| 문법 | 정답 변경, 출제 포인트, 정오답 해설 |
| 빈칸/논리 | 빈칸 타게팅, 근거, 보기 판단 |
| 대의파악 | 지문 분석, 정답 해설, 오답 소거 |
| 삽입 | 정답 해설, 위치별 설명 |
| 순서 | 1열, 분기점, 결론 |
| 어휘적절성 | 핵심 주제, 각 보기 해설 |

### 4. 번역 토글 기능 테스트

1. 설정에서 "번역 표시" 토글 찾기
2. 토글 ON: 지문 번역 + 선지 번역 표시
3. 토글 OFF: 번역 숨김
4. 기본값: ON (표시)

### 5. 지문 그룹핑 확인

1. 같은 지문을 가진 연속 문제 2개 입력
2. 해설지에서 지문이 한 번만 표시되는지 확인
3. 두 문제의 해설이 함께 표시되는지 확인

### 6. PDF 다운로드 테스트

1. PDF 다운로드 버튼 클릭
2. 10초 이내 생성 완료 확인 (30문제 기준)
3. A4 크기 레이아웃 확인
4. 실제 인쇄하여 텍스트 잘림 없음 확인

## 문제 해결

### 번역이 표시되지 않는 경우

1. AI 해설이 생성되었는지 확인
2. `showTranslation` 상태가 `true`인지 확인
3. `passageTranslation` 필드에 값이 있는지 확인

### PDF 생성이 느린 경우

1. 문제 수 확인 (100문제 이상 시 성능 저하 가능)
2. 브라우저 개발자 도구에서 에러 확인
3. 메모리 사용량 확인

### 레이아웃이 깨지는 경우

1. 폰트 크기 설정 확인 (0.85x ~ 1.2x)
2. Pantone 컬러 팔레트 적용 확인
3. A4PageLayout 컴포넌트 정상 동작 확인
